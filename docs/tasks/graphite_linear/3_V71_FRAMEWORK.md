## 3.41 2 When many branches share missing fixes

If many branches from the same target/root are missing the same important fix or conflict with the same newer baseline, prefer diagnosing root-level remediation.

The system should recommend one of:

```text
refresh target root first
create clean integration base
recreate branches from refreshed target
linearise each branch individually
leave affected branches in triage
exclude affected branches
```

It should not recommend blindly restacking every branch independently if the root itself is stale.

Example recommendation:

```json
{
  "target_root": "scientific",
  "diagnostic_category": "shared_root_staleness",
  "affected_branch_count": 12,
  "recommended_action": "root_refresh_decision_required",
  "why": [
    "multiple branches from scientific miss the same upstream fixes",
    "restacking each branch independently would duplicate conflict resolution",
    "target-level remediation may reduce repeated conflicts"
  ],
  "next_command": "python .graphite-agent/tools/root_questions.py --target scientific"
}
```
