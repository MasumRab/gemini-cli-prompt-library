# Agent Guidance

This file provides standardized guidance for any development agent (Claude, Gemini, Cursor, Vibe, etc.) working on this repository.

---

## Universal Coldstart Procedure

Run these commands in order on any fresh session:

```bash
# Step 1: Check repository state (ALWAYS FIRST)
git status

# Step 2: Check for stacking context (if present)
cat .graphite-agent/README.md 2>/dev/null || echo "No local stacking context"

# Step 3: Check for generated GitHub context
cat docs/ACTIVE_CONTEXT.md 2>/dev/null || echo "No active context file"

# Step 4: Review branch state
Git branch -a
git log --oneline -5
```

### Conditional (Local Workspace Only)
| File | Purpose | Scope | Availability |
|------|---------|-------|--------------|
| `.graphite-agent/README.md` | Stacking plan & branch state | stacking | Local only |
| `.graphite-agent/outputs/*.json` | Stacking configuration | stacking | Local only |
| `.graphite-agent/tools/*.py` | Stacking tools | stacking | Local only |

### Discovery Commands
```bash
# Find all markdown context files
find . -name "*.md" -path "*/docs/*" -o -name "*CONTEXT*" -o -name "*AGENTS*"

# Find all JSON configuration files
find . -name "*.json" -path "*/outputs/*" -o -name "*config*" -o -name "*state*"

# Find stacking-related files
find . -type d -name ".graphite-agent" -exec ls -la {} \; 2>/dev/null
```

## Project-Specific Context

### Graphite-Agent Stacking
If `.graphite-agent/` directory exists, the stacking plan is documented in [.graphite-agent/README.md](.graphite-agent/README.md).

**Quick Check:**
```bash
# If stacking context exists locally
if [ -d ".graphite-agent" ]; then
    echo "Stacking Plan:"
    cat .graphite-agent/README.md
    echo -e "\nKey States:"
    cat .graphite-agent/outputs/stack_order.json | jq '{root_health, execution_allowed}'
    cat .graphite-agent/outputs/execution_plan.json | jq '.execution_queue[] | {branch, status, action}'
fi
```

### GitHub Context
Generated by `scripts/update_active_context.py`:
```bash
# Generate fresh context (requires GITHUB_TOKEN)
python scripts/update_active_context.py

# View current context
cat docs/ACTIVE_CONTEXT.md
```

## Common Workflows

### 1. Starting New Work
```bash
# Always begin with discovery
git status
Git pull --rebase  # or merge, depending on strategy

# Check for existing context
if [ -f "docs/ACTIVE_CONTEXT.md" ]; then
    cat docs/ACTIVE_CONTEXT.md
fi

if [ -f ".graphite-agent/README.md" ]; then
    cat .graphite-agent/README.md
fi
```

### 2. Resolving Git Divergence
```bash
# Check divergence
git status
git log --oneline --left-right origin/$(git branch --show-current)...HEAD

# If .graphite-agent exists, check for stacking implications
if [ -d ".graphite-agent" ]; then
    echo "Check stacking plan for branch relationships"
    cat .graphite-agent/outputs/stack_order.json
fi
```

### 3. Before Committing
```bash
# Check for merge conflicts
git status

# Validate stacking if applicable
if [ -d ".graphite-agent" ]; then
    python .graphite-agent/tools/validate_stack_order.py
    python .graphite-agent/tools/validate_plan.py
fi

# Check GitHub context
if [ -f "docs/ACTIVE_CONTEXT.md" ]; then
    echo "Check for locked files in active PRs"
    grep -E "locked|\|" docs/ACTIVE_CONTEXT.md || echo "No locked files"
fi
```

## Error Handling

### Missing Files
If context files are missing:
1. Check if `.graphite-agent/` directory exists
2. Check `docs/ACTIVE_CONTEXT.md`
3. Fall back to standard git commands
4. Generate fresh context if possible

### Broken References
All references to `.graphite-agent/*` files are **gracefully degraded**:
- Relative links in markdown will show as broken (expected - directory may not exist)
- Scripts checking for files use `2>/dev/null` to suppress errors
- Agents should check for directory existence before accessing

### Authentication Issues
```bash
# GitHub API tokens
export GITHUB_TOKEN="your_token_here"

# If missing token
python scripts/update_active_context.py || echo "GitHub context unavailable"
```

## Agent-Specific Notes

### For All Agents
- Start with `git status` - always
- Check `.graphite-agent/README.md` if it exists
- Check `AGENTS.md` for updates
- Check `docs/ACTIVE_CONTEXT.md` for PR context

### Convention
- File references: Use relative paths from repo root
- Optional files: Always check existence before reading
- State files: Prefer JSON for machine readability
- Documentation: Prefer markdown for human readability
